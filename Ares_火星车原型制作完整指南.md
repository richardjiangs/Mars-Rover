# Operation MROS

IMPORTANT

## 🛒

为了实现复杂的功能，我们将采用“双脑”架构：一个ESP32负责控制和传感器，另一个ESP32-CAM专门负责图像处理和传输。

### 1. 核心大脑与视觉

- 1x ESP32 开发板 (NodeMCU): $6 (主控芯片，负责电机、超声波、陀螺仪、休眠逻辑)
- 1x ESP32-CAM 模块 (带OV2640摄像头): $7 (负责拍照、Wi-Fi图传)
- 1x SG90 微型舵机: $2 (用于搭载ESP32-CAM，实现360°旋转扫描)

### 2. 动力与底盘系统

- 6x TT 直流减速电机 + 配套越野轮胎: $15-$25 (V1 可先用便宜 1:48 验证结构；V2 推荐统一升级到 1:90，重车或强调爬坡可考虑 1:120)
- 2x L298N 电机驱动模块: $6 (一个控制左侧3个轮子，一个控制右侧3个轮子)
- 底盘与悬挂材料: $5-$15 (推荐用 PVC 管、木条、亚克力条或铝方管制作 Rocker-Bogie 摇臂-转向架；铰链用螺丝、垫片和防松螺母，不要粘死)

### 3. 导航与避障传感器

- 3x HC-SR04 超声波传感器: $5 (分别安装在正前方、左前和右前，用于防撞和自动驾驶寻路)
- 1x MPU6050 陀螺仪/加速度计模块: $3 (极其重要：用于修正电机转速差，确保能在5米内走绝对直线)
- 1x 光敏电阻模块 (LDR): $1 (用于检测光照强度，决定是否唤醒火星车)

### 4. 能源与供电系统

- 1x 5V 小型太阳能电池板 (约130x150mm): $5 (安装在车顶)
- 1x 2S BMS 锂电池保护板: $2-$4 (用于 2 节 18650 串联电池的过充、过放和短路保护；TP4056 只适合单节锂电池，不作为本项目主方案)
- 2x 18650 锂电池 + 1x 双节电池盒: $10 (主电源)
- 1x LM2596 降压模块 (DC-DC): $3 (将电池的7.4V降压至5V，供ESP32和舵机使用)

### 5. 杂项与结构件

- 被动式岩石收集器: $0 (DIY：在车尾底部倾斜安装一个切开的硬塑料瓶或金属小铲，行驶时自然兜住小石块)
- 保温与热控材料: $3-$8 (EVA 泡棉、薄海绵、泡沫板、绝缘胶带、可选 DS18B20 温度传感器；用于实现 -20°~50°目标的保温与温度保护演示)
- 面包板、杜邦线 (公对公/公对母/母对母)、热熔胶枪、M3螺丝螺母、扎带、独立电源开关。

## 🛠️


### 步骤 1：搭建“摇臂-转向架”底盘与机械减震系统

本项目的“机械减震”不采用汽车弹簧减震器，而采用火星车常见的 `Rocker-Bogie / 摇臂-转向架`。它的核心不是“弹簧变软”，而是让每侧 3 个轮子通过两个活动铰链上下摆动，使轮子遇到障碍时车身不会被直接顶起。

#### 1.1 一侧三轮的结构

每一侧由一根长摇臂和一根小转向架组成：

```text
一侧侧视图：

                 中央车身盒 / 主控舱
             ┌────────────────────┐
             │ ESP32 / 电池 / 驱动 │
             └─────────O──────────┘
                       │ 主铰链 A
                       │
前轮电机+轮子  ○────────┘────────O  二级铰链 B
                                  │
                           中轮 ○─┴─○ 后轮
                              小转向架 Bogie
```

拆开看就是：

```text
长摇臂 Rocker：
前轮电机位 ───── 主铰链 A ───── 二级铰链 B

小转向架 Bogie：
中轮电机位 ───── 二级铰链 B ───── 后轮电机位
```

- 前轮压到障碍物时，长摇臂绕主铰链 A 转动。
- 中轮或后轮压到障碍物时，小转向架绕二级铰链 B 转动。
- 这样可以让六个轮子尽量保持接地，把冲击分散到多个轮子和连杆上。

#### 1.2 推荐尺寸（配 65mm 左右 TT 越野轮）

| 零件 | 建议尺寸 | 说明 |
| --- | --- | --- |
| 中央车身盒 | 约 100–160mm 长 | 放电池、ESP32、L298N、LM2596 |
| 每侧长摇臂 | 约 220–260mm | 前端固定前轮，后端连接小转向架 |
| 每侧小转向架 | 约 140–170mm | 两端固定中轮、后轮，中间接长摇臂 |
| 主铰链 A | M4/M5/M6 螺丝 + 垫片 | 必须能转动，不能粘死 |
| 二级铰链 B | M4/M5 螺丝 + 垫片 | 必须能转动，不能粘死 |

孔位示意：

```text
长摇臂：
前轮电机位        主铰链 A              二级铰链 B
   O──────────────O────────────────────O

小转向架：
中轮电机位        二级铰链 B            后轮电机位
   O──────────────O────────────────────O
```

#### 1.3 制作步骤

1. 先做中央车身盒：用 ABS 接线盒、塑料盒、木板盒或亚克力盒作为主控舱，两侧各加一块侧板，用来安装主铰链 A。
2. 做左右两根长摇臂：材料可用 10x10mm 铝方管、木条、PVC 扁条、亚克力条或金属尺。左右必须尽量等长。
3. 做左右两个小转向架：中间打孔接二级铰链 B，两端分别固定中轮和后轮电机。
4. 铰链处用“螺丝 + 垫片 + 螺母”，不要用热熔胶粘死。拧紧后稍微回松，让摇臂能自由摆动但不要明显晃动。
5. 每个 TT 电机最好先固定在小电机座或小木片上，再用扎带穿孔固定到摇臂末端，最后少量热熔胶防滑。不要只靠一大坨热熔胶承重。
6. 在摇臂可能撞到车身的位置贴海绵、EVA 或泡沫块作为限位缓冲。也可以用橡皮筋轻轻拉住摇臂，让它回到中间角度，但橡皮筋不要作为主要承重件。
7. 电机线必须留出弯折余量，避免摇臂上下摆动时把线拉断。

#### 1.4 第一版简化方案

如果制作难度太高，可以先做“简化 Rocker-Bogie”：

- 不做每个轮子的独立转向；
- 六个轮子方向固定；
- 用左右两侧差速 / 坦克转向完成转弯；
- 先用便宜的 `1:48 TT 电机` 验证结构和接线，成功后再统一换成 `1:90` 或 `1:120`。

注意：六个驱动电机必须尽量使用同一减速比，不要左右混用不同减速比，否则会明显跑偏。


### 步骤 2：连接动力与供电网络

- 串联电池：将两节18650电池放入电池盒（提供约7.4V电压）。
- 电源分配：
  - 将7.4V直接连接到两个 L298N 驱动模块的 12V 输入 引脚（提供充足动力）。
  - 将7.4V连接到 LM2596 降压模块的输入端，调节降压模块上的电位器，用万用表测出精确的 5.0V 输出。
  - 将降压后的 5V 供给 ESP32、ESP32-CAM 和所有的传感器。
- 接线电机：左侧3个电机的正负极并联，接入第一个 L298N；右侧3个电机并联，接入第二个 L298N。这样你可以像控制坦克一样，通过左右履带的速度差来转向（甚至原地打转）。

### 步骤 3：部署“大脑”与传感器

- 直行保障 (MPU6050)：将其固定在车体正中央。代码中需要写一个 PID 控制器，如果陀螺仪发现车头偏向左，就稍微降低右侧电机的速度，强制车身走直线。
- 防撞系统 (超声波)：在车头品字形安装3个HC-SR04。代码逻辑：当前方距离 < 20cm 时，停止 -> 读取左右超声波数据 -> 向距离大的一侧转向。
- 视觉云台 (ESP32-CAM)：将 SG90 舵机固定在车顶，把 ESP32-CAM 粘在舵机臂上。


### 步骤 3.5：制作保温与温度保护系统（-20°~50°目标）

本项目的温度目标应分成两层理解：

- **模型演示层面**：用保温盒、温度传感器和程序保护逻辑，展示火星车具备“低温休眠、高温保护”的工程设计。
- **真实工程层面**：普通 TT 电机、18650 电池、ESP32 模块并不等于真的可以长期在 -20°C 到 50°C 环境中满负荷工作。因此本项目不建议使用没有温控的电阻加热片或把锂电池放进极端环境里测试。

#### 3.5.1 被动保温盒结构

把电池、ESP32、ESP32-CAM、传感器接口尽量放进中央主控舱内，做成“夹层保温盒”：

```text
外部环境
  │
  ▼
┌────────────────────┐
│ 外壳：ABS盒/塑料盒   │  防撞、防灰尘
│ ┌────────────────┐ │
│ │ 保温层：EVA/海绵 │ │  减少热量流失
│ │ ┌────────────┐ │ │
│ │ │ 空气层/泡沫 │ │ │  形成隔热缓冲
│ │ │ ESP32/电池  │ │ │  核心电子舱
│ │ └────────────┘ │ │
│ └────────────────┘ │
└────────────────────┘
```

可用材料：EVA 泡棉、薄海绵、泡沫板、瓦楞纸板、热缩管、绝缘胶带、少量铝箔胶带或急救毯反光膜。铝箔只能贴在外层或夹层，不能接触电路板焊点和电池极片，防止短路。

#### 3.5.2 冷环境策略

- 电池和主控板放在保温盒中心，不要贴在底盘外侧。
- L298N 会发热，不能完全包死；可让散热片靠近通风孔或外壳内壁。
- 低温时先进入 `COLD_START`：上电后原地等待 30–60 秒，让电子舱内部温度稳定，再低速行驶。
- 当温度过低时，火星车进入 `COLD_SLEEP`：关闭电机，只保留主控低功耗监测，等待光照或温度恢复。

#### 3.5.3 热环境策略

- 电子舱内部温度超过 45°C 时降低电机功率，减少 ESP32-CAM 连续图传。
- 温度超过 50°C 时进入 `THERMAL_PAUSE`：停车、停止拍摄、等待降温。
- 外壳顶部可使用浅色材料或反光膜；但太阳能板表面不能被覆盖。

#### 3.5.4 温度传感器选项

建议加一个可选温度传感器，便于在电脑端显示内部温度：

- 简单方案：`DS18B20` 数字温度探头，装在主控舱内部靠近电池的位置。
- 替代方案：`10k NTC` 温度模块，接 ESP32 的 ADC 输入脚。

推荐程序规则：

```text
inside_temp < -10°C  -> COLD_SLEEP，禁止电机运行
-10°C ~ 0°C          -> COLD_START，只允许低速短时运行
0°C ~ 45°C           -> NORMAL，正常运行
45°C ~ 50°C          -> HOT_LIMIT，降低速度和拍摄频率
inside_temp > 50°C   -> THERMAL_PAUSE，停车降温
```

#### 3.5.5 安全边界

- 不要用裸电阻丝、没有温控的加热片直接贴在 18650 电池上。
- 不要让铝箔、金属胶带、螺丝接触电池正负极或电路板焊点。
- 不要把 L298N、LM2596 完全包进厚海绵里运行，防止自身过热。
- 真正低温测试时，不建议把锂电池放进冰箱或冷冻环境反复测试；课堂模型可用冰袋放在盒子外侧进行短时间模拟，并全程人工看护。


### 步骤 4：实现 3D 和 360° 摄影“黑科技”

预算有限买不起3D镜头，我们用软件弥补：

- 360°全景逻辑：主控板发送指令，让舵机从 0° 转到 180°，每转 30° 让 ESP32-CAM 拍一张照并传给电脑。电脑端使用 Python 脚本（利用 OpenCV 库）自动将图片拼接成全景图。
- 3D照片逻辑：利用双目视差原理。让火星车在原地拍一张照 (左眼视角) -> 火星车绝对直线向前移动 6 厘米 -> 再拍第二张照 (右眼视角)。将两张图片传给电脑，可以通过软件合成为红蓝3D图片或立体图。

### 步骤 5：编写“3个月火星生存”休眠代码逻辑

由于不能真的跑3个月，我们可以用代码模拟时间流逝（例如现实中1天等于火星上1周）：

- 光控唤醒：将光敏电阻 (LDR) 连接到 ESP32 的唤醒引脚。
- 深度休眠 (Deep Sleep)：写一段逻辑，当 LDR 检测不到光（黑夜），ESP32 进入 Deep Sleep 模式（功耗仅几微安），同时关闭电机供电。
- 充电与唤醒：天亮时，太阳能板通过 2S BMS 为 2 节串联 18650 电池补能。LDR 电阻值变化，唤醒 ESP32。
- 最后10天的“冲刺模式”：
  - ESP32 内部使用 RTC (实时时钟) 内存记录自己醒来的次数。
  - 假设设计为唤醒 80 次后（代表进入最后生命周期），触发 Final_Mission_Mode()。
  - 在此模式下，火星车无视光照和电量警告，全速运行，高频调用 ESP32-CAM 拍摄所需的50张全景照片和100张3D照片素材，并通过超声波和陀螺仪数据在电脑端绘制出其走过的地图路线，直到电池彻底耗尽，光荣完成使命。

安全警告: 处理18650锂电池时请注意正负极，切勿短路。调试电机时请先将火星车架空，防止意外跑飞。
