# Ares V3 GPIO Map（全车唯一权威引脚表）

> 本表取代散佚的 `⭐Ares Dual-Brain GPIO Map.docx`，并按 V3 基线（`hardware/drawings/线路图.jpg` 手绘接线）重建。
> 引脚改动必须先改本表，再同步 `firmware/commander/config.h` 与 `firmware/cam/config.h`。
> 来源标注：【手绘】= 线路图.jpg 明确画出；【V2.2】= 沿用旧 Wiring Chart；【推断】= 为落地补齐的工程推断；【待确认】= 需要实物验证或用户拍板。

## 1. Commander ESP32（NodeMCU 38pin）

### 1.1 电机驱动（L298N）

| 功能 | ESP32 GPIO | L298N 引脚 | 来源 |
| --- | --- | --- | --- |
| 左侧 3 电机 正转 | GPIO 16 | IN1 | 【手绘】"左正D16" |
| 左侧 3 电机 反转 | GPIO 17 | IN2 | 【手绘】"左反D17" |
| 右侧 3 电机 正转 | GPIO 18 | IN3 | 【手绘】"右正D18" |
| 右侧 3 电机 反转 | GPIO 19 | IN4 | 【手绘】"右反D19" |

- 调速方式：固件直接在 IN1–IN4 上输出 LEDC PWM（正转脚给占空比、反转脚拉低），不占用 ENA/ENB。ENA/ENB 保持跳线帽常使能。【推断，依据 `docs/Ares_Control_Architecture.md` §9.3】
- L298N `+12V` 接 7.4V 总线，`GND` 接共地，板上 5V 稳压跳线保持默认（不从 L298N 取 5V 给逻辑板）。【手绘】

⚠️ **单 L298N 电流风险**：每通道并联 3 个 TT 电机，堵转时总电流可能达 3–6A，超过 L298N 单通道约 2A 的持续能力。缓解措施：
1. 固件内置软启动斜坡（默认 ≤ 每 20ms 增加 5% 占空比）和最高占空比限幅；
2. 避免长时间堵转（避障逻辑遇阻即停）；
3. 若实测驱动板过热/掉电，升级为双 L298N：左组接第 1 块的 IN1/IN2+OUT1/OUT2，右组接第 2 块的 IN3/IN4+OUT3/OUT4，**固件引脚完全不变**。【待确认：实车电流实测】

### 1.2 超声波（3 × HC-SR04，独立三向 = AUTO-L2）

| 位置 | Trig | Echo | 来源 |
| --- | --- | --- | --- |
| 前方 Front | GPIO 13 | GPIO 14 | 【手绘】D13/D14 一组 |
| 左前 Front-Left | GPIO 27 | GPIO 26 | 【手绘】D27/D26 一组 |
| 右前 Front-Right | GPIO 33 | GPIO 32 | 【手绘】D33/D32 一组 |

- 手绘图能确认三组引脚配对，但**哪组对应哪只探头未标注**，以上 F/L/R 分配为【推断】，装车后如不符只需改 `config.h`，不必改线。
- ⚠️ **Echo 是 5V 输出，必须分压**：每路 Echo 串 1kΩ 后对 GND 接 2kΩ，从分压点接 ESP32（3.3V ≈ 5V × 2/3）。HC-SR04 的 VCC 仍接 5V 总线。【推断，安全底线】
- 三路轮询测距（间隔 ≥ 30ms），防止互相串扰。【推断】

### 1.3 I2C（MPU6050）

| 功能 | ESP32 GPIO | MPU6050 | 来源 |
| --- | --- | --- | --- |
| SDA | GPIO 21 | SDA | 【V2.2/ESP32 标准】 |
| SCL | GPIO 22 | SCL | 【V2.2/ESP32 标准】 |

- ⚠️ 手绘图标注为 `SCL–D21、SDA–D22`，与 ESP32 标准（SDA=21/SCL=22）**相反**。固件默认按标准；若实物按手绘接了线，改 `config.h` 的 `PIN_I2C_SDA/PIN_I2C_SCL` 两个常量即可，无需改线。【待确认：以实物接线为准】
- MPU6050 VCC 接 3V3（模块带稳压时接 5V 也可，手绘接 3V3），必须水平安装、线长 < 10cm、紧贴洞洞板刚性固定。【手绘】

### 1.4 云台舵机与相机链路

| 功能 | ESP32 GPIO | 对端 | 来源 |
| --- | --- | --- | --- |
| 云台 SG90 PWM | GPIO 4 | SG90 橙线 | 【V2.2 沿用；手绘缺失但任务需要】 |
| UART2 TX | GPIO 25 | ESP32-CAM GPIO 3 (U0RXD) | 【推断：V2.2 的 16/17 已被电机占用，重映射】 |
| UART2 RX | GPIO 23 | ESP32-CAM GPIO 1 (U0TXD) | 【推断，同上】 |

- UART2 参数：115200 8N1，ASCII 行协议（见 `docs/Ares_Control_Architecture.md` §11.2）。
- SG90 电源接 5V 总线；6 只舵机中仅 1 只入基线（云台），折叠翼舵机随太阳能取消。【V3】

### 1.5 可选传感器（基线预留，未购/未接不影响运行）

| 功能 | ESP32 GPIO | 说明 | 来源 |
| --- | --- | --- | --- |
| LDR 模拟量 AO | GPIO 34（输入专用，ADC1_CH6） | 光强读数，判断昼/夜 | 【推断】 |
| LDR 数字量 DO | GPIO 35（输入专用，RTC GPIO） | 深度休眠 ext0 唤醒源 | 【推断】 |
| DS18B20 DATA | GPIO 15 + 4.7kΩ 上拉到 3V3 | 舱内温度；V2.2 的 GPIO27 已被超声波占用 | 【推断】 |
| 电池电压检测 | GPIO 36（输入专用，ADC1_CH0） | 7.4V 经 100kΩ/33kΩ 分压 ≈ 满电 2.1V | 【推断：遥测 battery_v 需要；不接则遥测报 0】 |

### 1.6 Commander 电源

| 引脚 | 接法 | 来源 |
| --- | --- | --- |
| VIN | 5V 总线（LM2596 OUT+） | 【手绘】 |
| GND | 共地总线 | 【手绘】 |

### 1.7 引脚占用一览（Commander）

```
已用   : 4(舵机) 13,14(超声F) 16,17,18,19(电机) 21,22(I2C) 23,25(UART2) 26,27(超声L) 32,33(超声R)
```

## 2. ESP32-CAM（AI-Thinker）

| 功能 | 引脚 | 说明 |
| --- | --- | --- |
| 5V / GND | 5V 总线 / 共地 | 【手绘】直接 5V 供电更稳（3V3 供电易复位） |
| U0TXD (GPIO 1) | → Commander GPIO 23 | UART 从协议上行 |
| U0RXD (GPIO 3) | ← Commander GPIO 25 | UART 从协议下行 |
| GPIO 0 | 烧录时接 GND，运行时悬空 | 烧录模式选择 |
| GPIO 4 | 板载闪光灯 LED（固件可用作拍照补光） | 板内固定 |
| GPIO 33 | 板载红色状态灯（低电平点亮） | 板内固定 |
| 摄像头排线各脚 | AI-Thinker 标准分配 | 见 `firmware/cam/camera_pins.h` |

⚠️ CAM 用 U0（烧录串口）兼作与主控的链路：**烧录/接串口监视器前必须先断开与 Commander GPIO25/23 的两根线**，否则烧录失败。运行时再接回。【推断，需实机遵守】

## 3. 供电拓扑（V3）

```
2×18650(7.4V) → 2S BMS(B+/B-) → P+/P- → 船型/钮子总开关 → [7.4V 总线]
[7.4V 总线] → L298N +12V             （电机动力）
[7.4V 总线] → LM2596 IN+ → OUT+ 调至 5.00V → [5V 总线]
[5V 总线]  → ESP32 VIN、ESP32-CAM 5V、HC-SR04×3 VCC、SG90、（LDR 模块）
[GND]      → 全系统共地（BMS P-、LM2596、L298N、两块板、所有传感器）
充电：取出电池用 2S 专用充电器，或经 BMS 的充电口用 8.4V CC/CV 适配器（太阳能已取消）
```
