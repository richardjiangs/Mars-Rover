# Ares V3 决策记录（Decision Log）

> 记录两代设计（V2.2 文档系 vs V3 手绘系）冲突的裁决结果与理由。
> 每条标注：【文档确认】= 有仓库资料明确依据；【工程推断】= 为落地补齐；【待确认】= 需实物验证。
> 基线切换由用户于 2026-07 拍板：**以 V3 手绘最新版为项目基线**。

## D1. 项目基线 = V3 手绘版 【文档确认 + 用户拍板】

- 依据：`hardware/drawings/总体设计.jpg`、`hardware/drawings/线路图.jpg`、`docs/楼层布局.md`、`docs/2米高空坠落生存与自恢复方案.md`、`hardware/cad/*.stl` 相互吻合（主舱 190×149mm 与 主舱.stl 实测包络一致），且 `hardware/drawings/部件.jpg` 显示实际采购与 V3 一致（无太阳能板）。
- 结论：V2.2 文档（完整指南、BOM、Wiring Chart）降级为历史参考；冲突时 V3 优先。

## D2. 取消太阳能板，改外部充电 【文档确认】

- 依据：`docs/楼层布局.md` 天台层明确写"取消太阳能板：释放全部车顶空间"；部件照片无太阳能板；车顶空间让位于甲虫穹顶（跌落自翻正）。
- 影响：折叠翼舵机（V2.2 GPIO 13/12）取消；13 号脚改用于超声波；"100 天生存"演示改为：LDR（可选）+ 深度休眠 + RTC 唤醒计数模拟昼夜，充电由人工用 2S 充电器完成。
- 风险：无自主补能，长时演示需人工换电/充电。【待确认：演示时长安排】

## D3. 单块 L298N 驱动六电机 【文档确认 + 风险记录】

- 依据：线路图.jpg 只画一块 L298N（OUT1/2 左组、OUT3/4 右组）；部件照片只有一块驱动板。
- 风险：每通道 3 只 TT 电机并联，堵转电流可能超 L298N 单通道 ~2A 持续能力。
- 缓解：固件软启动斜坡 + 占空比限幅 + 遇障即停；实测过热则升级双 L298N（接法见 `hardware/Ares_V3_GPIO_Map.md` §1.1，固件引脚不变）。【待确认:T1 架空电流实测】

## D4. 超声波 = 3 只独立三向感知（AUTO-L2） 【文档确认】

- 依据：线路图.jpg 画了 3 组独立 Trig/Echo（13/14、27/26、33/32），与 `docs/Ares_Control_Architecture.md` §9.2 推荐的 AUTO-L2 一致，取代 V2.2 Wiring Chart 的并联方案（AUTO-L1）。
- F/L/R 与三组引脚的对应关系图中未标注，在 `firmware/commander/config.h` 中可调。【工程推断】
- Echo 必须 1k/2k 分压后接 ESP32。【工程推断，安全】

## D5. 电机引脚 = GPIO 16/17/18/19 【文档确认】

- 依据：线路图.jpg 标注"左正D16 左反D17 右正D18 右反D19"。
- 影响：V2.2 的 UART2（16/17）必须重映射 → 见 D7。

## D6. I2C 方向：代码按 SDA=21 / SCL=22 【工程推断，待确认】

- 冲突：线路图.jpg 写 `SCL–D21、SDA–D22`，与 ESP32 惯例（SDA=21/SCL=22）及 V2.2 Wiring Chart 相反。
- 裁决：固件默认 ESP32 标准方向；若实物按手绘接线导致 MPU6050 初始化失败，把 `config.h` 里 `PIN_I2C_SDA/PIN_I2C_SCL` 互换即可（免改线）。
- 附带约束（手绘明确）：I2C 线长 < 10cm、MPU6050 水平刚性固定在 F2 洞洞板上。

## D7. UART2↔CAM 重映射为 TX2=25 / RX2=23 【工程推断】

- 原因：V2.2 用的 16/17 已被电机占用（D5）；手绘图未画 UART 链路，但 `docs/Ares_Control_Architecture.md` §11.2 的任务级命令链路（PING/CAPTURE/ACK/DONE）需要它。
- CAM 侧复用 U0（GPIO1/3）：烧录 CAM 前必须断开这两根线。
- 备选：若 25/23 与后续扩展冲突，ESP32 UART 可映射到任意空闲脚，改 config 即可。

## D8. 充电保护 = 2S BMS，不用 TP4056 【文档确认，维持既有裁决】

- 依据：原 AGENTS.md、BOM、Wiring Chart 一致；TP4056 仅适用单节锂电池。

## D9. 云台舵机保留 GPIO 4 【工程推断】

- 手绘图没画舵机，但任务目标（360° 全景 = 舵机 0–180° 分步扫描）与已购 SG90 都需要它；V2.2 分配的 GPIO 4 在 V3 引脚表中无冲突，沿用。
- 360° 覆盖：舵机只有 180°，剩余角度靠车体原地自转 180° 后再扫一轮补齐。【工程推断，符合"用自身旋转代替 360° 相机"的任务描述】

## D10. LDR / DS18B20 / 电池电压检测 = 可选件 【工程推断】

- 部件照片中未见 LDR 模块与 DS18B20；固件按"可选"实现（config 开关，未接不影响运行）：
  - LDR：AO→GPIO34（光强遥测/休眠判据）、DO→GPIO35（深睡唤醒）；未接时休眠演示用定时器唤醒替代。
  - DS18B20：GPIO15；未接时热控状态机静默停用（thermal 遥测报 N/A）。
  - 电池电压：100k/33k 分压→GPIO36；未接时 battery_v 报 0。
- 建议补购（见 `docs/下一步你要做什么.md`）。

## D11. 机械臂 = 第二阶段扩展 【文档确认】

- 依据：`docs/Ares_Control_Architecture.md` §12：影响预算/重心/爬坡/跌落/续航，不进当前基线；结构可预留安装面。

## D12. 电脑端形态 = 网页控制台 + Python 脚本 【用户拍板】

- dashboard/index.html 直连 Commander WebSocket（遥测/遥控/E-STOP/任务），照片经 CAM→HTTP→`vision/receive_server.py`；拼接、3D、地图由 Python 离线处理。
- 固件工程形式 = Arduino IDE（用户拍板）；仓库组织 = 全面重组（用户拍板）。

## D13. 温度演示的安全边界 【文档确认，维持】

- 只做被动保温 + 温度监测 + 程序保护（五态机）；不加无温控加热片；不把锂电池放入冰箱/冷冻环境测试。

## D14. 打印结构件全套参数化重建（2026-07） 【文档确认 + 工程推断】

- 触发：用户上传的 9 个 `*_ARES_FIXED_V9.stl` 经核查存在系统性错误——多壳体非水密/
  法线翻转（切片会破洞）、铰链孔径全为 5.5（与螺丝清单的 M4 二级铰链冲突）、六个电机位
  无任何螺孔、顶盖角孔与主舱角柱不对位（±80/3 vs +74/-86/±60）、主舱角柱不对称、
  轴座为方槽非 M5 孔、缺超声波支架。
- 重建：`hardware/cad/build_parts.py` 按 螺丝清单 + Footprint + 楼层布局 + 总体设计
  参数化生成 13 件水密实体，输出到 `Print required/`（6 盘 ≤215×240，平躺免支撑）。
- 裁决的尺寸修正：
  - 主舱高 75 → **90**（楼层叠高 88 放不进 75；总体设计.jpg 本来就标 9cm）。
  - 二级铰链 B 孔 Φ5.5 → **Φ4.5**（M4×20，按螺丝清单）。
  - 电机站 = Φ8 轴孔 + 外侧 Φ24 沉台（TT 轴仅 ~9mm，8mm 臂厚会吃光轴长）+
    2×M3 腰形槽（间距 17.6，容差 ±3）。
  - 钮子开关移至 B1 层高度（原高度本体撞 F1 楼层板），F1 后缘开 42×26 避让缺口。
  - 差动杆（NASA rocker-bogie differential）纳入基线：顶盖中心 M5 枢轴 +
    两端腰形槽连左右摇臂立柱；替代原"轴座+平衡杆"的破损设计。
  - 新增超声波三向支架（前直视+左右 32°，Φ16.4 泪滴孔，免支撑）。
  - 配重件取消：电池+电机已在低位，必要时车底加金属垫片（跌落方案的低重心要求
    由布局满足）。
- 校验：`hardware/cad/verify_fit.py` 39 项全 PASS（孔位对位≤0.35mm、孔径、
  打印件两两与器件干涉体积≈0、层间间隙、六轮共面）；装配渲染见
  `hardware/drawings/装配预览/`。
