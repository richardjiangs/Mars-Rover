# AGENTS.md

## 项目定位

本仓库用于设计一辆低成本火星车原型，当前阶段以方案设计、采购定义、接线规划、控制逻辑拆分为主。仓库现有内容主要是设计文档，后续新增内容默认应服务于“把这辆火星车从概念逐步落到可制作、可接线、可编程、可测试”的目标。

该项目当前的默认基线不是泛化的“小车项目”，而是基于现有文档中的 `PROJECT ARES / Operation MROS` 方案继续推进。


## 目标功能（所有文件提到的目标功能以此为主）
1. 六轮六驱，每个轮子不同速度实行转弯，总马力需约为8牛
2. 5m距离能基本能走直线
3. 能爬上25°的陡坡
4. 在2m高空坠落后仍能正常工作
5. 可自动驾驶
6. 可以使用机械臂采矿
7. 制作材料不超过150美元
8. 工作寿命为100天以上，并每天都会将当天所获数据转给PC端
9. 工作温度：-20°~50°（要有保暖措施）
10. 本火星车任务为：
    1. 拍摄到30张3D图片
    2. 拍摄到10张360°全景图片（用自身旋转让ESP32-CAM模块代替360°相机）
    3. 类似扫地机器人一样为预设区域制作一张地图
11. 机械 Rocker-Bogie 减震系统

## 当前项目基线

除非用户明确要求修改架构，否则后续工作应默认保持以下基线：

- 双脑架构：
  一个 `ESP32 NodeMCU` 负责主控；
  一个 `ESP32-CAM` 负责图像采集与传输。
- 六轮底盘：
  采用 `Rocker-Bogie / 摇臂-转向架` 思路，强调越障与抗跌落。
- 驱动方式：
  左右两侧各 3 个 TT 电机并联，由两块 `L298N` 分别驱动，按差速/坦克转向控制。
- 关键传感器：
  `MPU6050`、`HC-SR04`、`LDR`，以及用于相机扫描的 `SG90` 舵机。
- 能源系统：
  以 `2 x 18650` 电池、`LM2596` 降压、太阳能补能为主线。
- 热控系统：
  以中央被动保温盒、可选温度传感器、低温休眠和高温暂停为主线；不默认加入无温控主动加热片。
- 核心能力目标：
  直线行驶修正、避障、360°图像采集、伪 3D 视差拍摄、休眠/唤醒、最终任务模式、-20°~50°温度目标的模型化保护演示。

## 子系统职责划分

后续写方案、代码、测试说明时，默认按以下边界拆分：

- 主控 ESP32
  负责电机控制、姿态修正、避障逻辑、休眠唤醒、任务状态机、舵机动作、向相机板发指令。
- ESP32-CAM
  负责拍照、回传、必要的串口确认、可能的本地存储。
- 电脑端脚本
  负责全景拼接、伪 3D 图生成、轨迹/地图后处理、实验数据整理。

不要把所有职责混成单体设计；跨板通信接口要写清楚谁发命令、谁回确认、数据如何流转。

## 文档冲突处理规则

现有文档并非完全一致，后续工作必须显式处理冲突，不能静默混用。

已知冲突与默认处理方式：

- 充电方案冲突
  `Markdown 版完整指南` 提到 `TP4056`，但 `Markdown 版采购清单` 与 `Markdown 版 Wiring Chart` 明确围绕 `2S BMS` 和 `7.4V` 双节串联电池工作。
  在“2 节 18650 串联”这一前提下，默认优先采用 `2S BMS` 方案，不默认沿用 `TP4056`。
- 超声波方案冲突
  `Markdown 版完整指南` 描述了前/左前/右前 3 个独立超声波传感器参与选路；
  `Markdown 版 Wiring Chart` 则把 3 个 HC-SR04 并到同一组 `Trig/Echo`，更像一个总前保险杠。
  后续涉及避障设计、接线或代码时，必须先明确选择“独立三向感知”还是“并联单前向感知”。
- 太阳能翼与额外舵机
  `Markdown 版 Wiring Chart` 和 `GPIO Map` 中出现左右太阳能翼舵机等 v2.2 元素，但 `Markdown 版完整指南` 没有完整展开。
  默认将其视为高级扩展能力；若任务未明确涉及，可先按基础原型处理。

如果未来发现新的不一致：

- 优先采用“该领域最专门的文档”；
- 优先采用“与电压、电流、安全更一致的方案”；
- 在输出中明确写出：哪些是文档确认，哪些是推断，哪些是待用户拍板。

## 工作原则

- 不要脱离现有文档随意换平台、换主控、换供电电压等级。
- 做设计判断时，优先保留文档中的系统意图，再谈优化。
- 每次新增方案、说明、代码前，先确认它属于哪个子系统。
- 如果提出改动，必须说明它影响哪些器件、接线、代码模块和测试项。
- 如果信息不足，先补齐文档依据，再继续细化实现。

## 安全与工程底线

- 处理 `18650` 电池时，严禁短路，优先使用带保护的方案。
- 主控与传感器上电前，先确认 `LM2596` 输出稳定在 `5.0V` 左右。
- 所有模块默认共地；接地关系不清时，不要贸然出接线结论。
- 调试电机时优先让车轮离地，避免误动作导致跑飞。
- 涉及电池、充电、太阳能板接入的改动，默认按高风险任务处理。
- 涉及主动加热片、电阻丝、锂电池低温/高温测试的改动，默认按高风险任务处理；基线方案只采用被动保温和温度监测。
- Rocker-Bogie 铰链必须能转动，不能用热熔胶粘死；电机固定应优先使用螺丝/扎带承重，热熔胶只做辅助防滑。

## 建议的后续文件组织

仓库当前以文档为主。以后新增内容时，优先按下面的方向组织，而不是把不同类型文件混放在根目录：

- `docs/`
  设计说明、决策记录、实验结论、会议纪要。
- `hardware/`
  接线图、引脚表、BOM 衍生稿、结构草图、CAD 导出件。
- `firmware/commander/`
  主控 ESP32 固件。
- `firmware/cam/`
  ESP32-CAM 固件。
- `vision/`
  电脑端图像处理与数据处理脚本。
- `tests/` 或 `experiments/`
  标定脚本、单项验证、临时实验记录。

如果这些目录尚未创建，可以在真正需要时再创建，但新增文件时应尽量遵循这一组织方式。
